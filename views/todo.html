Fancy Todo List

Overview

What kind of developer am I without a todo-app? This is a Case Studies on Javascript Frameworks through A Fancy Todo App!

View it Here!

Goal/Objective

The goal of this project is to experiment with Vue.js for Web apps and React Native for mobile apps to create a simple app.

Solution

By creating a Todo-app, I will be able to use the concept of CRUD which is the essential of most apps. With this, I get to familiarize myself on how these frameworks handle this concept. I chose to do a web app and a mobile app to be able to know the process of how to do the same app on different platforms.

My Thoughts

Vue.js- Really amazing documentation which is helpful to someone with only HTML and Javascript experience. I really enjoyed how it was not as invasive as others, meaning, I can even just attach a small line of code to my HTML and that's it. However, because it is new, there aren't a lot of community support for it yet.

React Native- Since I jumped to React Native right away before learning React, it took me a while to just even set it up. The whole mobile development process was totally new to me. React Native has a lot of community support and flexibility. It's quite easy to get into it once you get the concept and very easy to understand as well. It also uses ES6/7 which made it a big learning tool for me. The documentation is a bit confusing and unclear at times. I also found out that some components might not work because of an update which apparently, happens a lot.

In Conclusion

This case studies project helped me be more familiar with Vue and React.While I can't really compare the two because of how different they are their purposes are, there are definitely a lot of freedom with React Native. However, I enjoy Vue's simplicity and non-invasive nature.Overall, I got to experience working on different work processes, so that was fun!
